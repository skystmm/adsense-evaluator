# Google AdSense 评估网站 - 产品需求文档

## 1. 产品概述

### 1.1 产品定位
一款帮助网站所有者评估其网站是否符合 Google AdSense 接入要求的工具，提供详细的评估报告、问题诊断和整改建议。

### 1.2 目标用户
- 想要申请 AdSense 的网站所有者
- 被 AdSense 拒绝的网站管理员
- 希望优化网站以符合 AdSense 要求的开发者

### 1.3 核心价值
- **快速评估**：输入 URL，快速获取 AdSense 接入可行性评分
- **问题诊断**：识别网站不符合 AdSense 要求的具体问题
- **AI 增强建议**：基于 AI 分析提供个性化的整改建议

## 2. 功能需求

### 2.1 网站评估功能
- 输入网站 URL
- 自动爬取和分析网站内容
- 输出评估报告，包括：
  - 整体评分（0-100 分）
  - 各项指标得分（内容质量、网站结构、用户体验、技术合规等）
  - 通过/不通过判断
  - 详细问题分析

### 2.2 问题诊断功能
- 根据 AdSense 官方拒接原因库进行匹配
- 识别具体问题：
  - 内容质量不足
  - 网站结构不清晰
  - 缺少必要页面（隐私政策、关于我们等）
  - 技术问题（加载速度慢、移动端不友好等）
  - 违反政策内容
- 提供针对性的整改建议

### 2.3 AI 增强功能
- 内容质量 AI 分析
- 个性化建议生成
- 整改优先级排序
- 预估通过概率

## 3. 技术架构

### 3.1 技术栈
- **前端**: Next.js 14 + React + TypeScript + TailwindCSS
- **后端**: Python FastAPI
- **AI 集成**: LLM API（用于内容分析和建议生成）
- **数据库**: PostgreSQL（生产）/ SQLite（开发）
- **部署**: Vercel (前端) + Railway/Render (后端)

### 3.2 系统架构
```
用户 → 前端 (Next.js) → 后端 API (FastAPI) → AI 服务
                              ↓
                        数据库 (PostgreSQL)
                              ↓
                        网站爬取服务
```

### 3.3 核心模块
1. **网站爬取模块**: 负责抓取和分析目标网站
2. **评估引擎**: 根据 AdSense 标准进行打分
3. **AI 分析模块**: 调用 LLM API 进行深度分析
4. **报告生成模块**: 生成可视化评估报告
5. **用户管理模块**: 用户认证和历史记录

## 4. 评估指标体系

### 4.1 内容质量（30 分）
- 原创性
- 内容深度
- 更新频率
- 专业性

### 4.2 网站结构（25 分）
- 导航清晰度
- 页面层次
- 内部链接
- 必要页面完整性

### 4.3 用户体验（25 分）
- 加载速度
- 移动端适配
- 可读性
- 广告位置合理性

### 4.4 技术合规（20 分）
- HTTPS 支持
- SEO 优化
- 隐私政策
- 联系方式

## 5. 开发计划

### 阶段 1 - 项目初始化
- 创建 GitHub 仓库

- 搭建基础项目结构

### 阶段 2 - 核心功能开发

- 评估打分系统

### 阶段 3 - 问题诊断
- 拒接原因库建设
- 诊断逻辑实现
- 整改建议生成

- 完整前端页面
- 用户认证系统
- 历史记录功能
- 部署上线

## 6. 成功标准

- 评估准确率 > 85%
- 用户满意度 > 4.0/5.0
- 响应时间 < 30 秒
- 支持并发用户 > 100

## 7. 风险与挑战

- Google AdSense 政策变化
- 网站爬取的技术限制
- AI 分析准确性
- 法律合规性
